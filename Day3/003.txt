1. 객체지향 프로그래밍

객체지향 프로그래밍은 데이터와 기능을 하나의 객체로 묶어서 프로그램을 구성하는 방식이다.
현실의 사물처럼 속성과 동작을 함께 표현할 수 있다.

2. 클래스와 객체

클래스는 객체를 만들기 위한 설계도이다.
객체는 클래스로부터 만들어진 실제 데이터이다.

예시
class Student:
    pass

student = Student()

3. 생성자와 소멸자

__init__
- 객체가 생성될 때 자동으로 실행된다.
- 객체의 초기 속성을 설정할 때 주로 사용한다.

__del__
- 객체가 삭제될 때 호출될 수 있다.
- 실제 코드에서는 직접 의존하기보다 리소스 관리를 명확하게 하는 편이 좋다.

4. 상속

상속은 부모 클래스의 기능을 자식 클래스가 물려받는 기능이다.
중복 코드를 줄이고, 기존 기능을 확장할 때 사용한다.

super()
- 부모 클래스의 메서드를 호출할 때 사용한다.

5. 다형성

다형성은 같은 이름의 메서드가 객체에 따라 다르게 동작하는 성질이다.
예를 들어 여러 클래스가 같은 speak() 메서드를 가지더라도 각 클래스가 다른 방식으로 구현할 수 있다.

6. 추상 클래스

추상 클래스는 반드시 구현해야 하는 메서드의 틀을 제공한다.
abc 모듈을 사용하면 자식 클래스가 특정 메서드를 구현하도록 강제할 수 있다.

7. 캡슐화와 접근 제한 관례

캡슐화는 객체 내부 데이터를 외부에서 함부로 바꾸지 못하도록 관리하는 방식이다.

Python의 접근 관례
- public: 일반 이름. 어디서든 접근 가능하다.
- _protected: 내부용이라는 의미를 가진 관례이다. 외부 접근은 가능하지만 권장하지 않는다.
- __private: 이름 맹글링이 적용되어 외부에서 직접 접근하기 어렵다.

핵심
클래스는 관련 있는 데이터와 기능을 묶고, 상속은 기존 기능을 재사용하며, 캡슐화는 데이터를 안전하게 다루기 위한 방법이다.
