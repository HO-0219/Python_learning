학습 주제
- 데코레이터
- 제너레이터

1. 데코레이터

데코레이터는 기존 함수를 감싸서 기능을 추가하거나 동작을 변경하는 함수이다.
기존 함수 코드를 직접 수정하지 않고 앞뒤에 공통 기능을 붙일 수 있다.

사용 형태
@decorator_name
def my_function():
    pass

자주 쓰는 경우
- 함수 실행 시간 측정
- 로그 출력
- 권한 검사
- 반복되는 전처리와 후처리

2. 제너레이터

제너레이터는 yield를 사용하여 값을 하나씩 반환하는 함수이다.
모든 데이터를 한 번에 메모리에 올리지 않고, 필요할 때마다 하나씩 만든다.

장점
- 메모리를 절약할 수 있다.
- 큰 데이터나 반복 흐름을 다룰 때 유용하다.
- next()로 값을 하나씩 꺼낼 수 있다.

핵심 차이
- return은 값을 반환하고 함수를 종료한다.
- yield는 값을 하나 반환한 뒤 함수 상태를 유지한다.

한 줄 요약
데코레이터는 함수를 감싸 기능을 확장하는 문법이고, 제너레이터는 값을 필요한 만큼 하나씩 만들어 메모리를 아끼는 문법이다.
